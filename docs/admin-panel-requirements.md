# Recta2 管理画面 機能要件まとめ

## 技術スタック

- **フロントエンド**: React Router v7 (Framework mode / SSR) + React 19 + TypeScript + Vite 7
- **UI**: Tailwind CSS 4 + shadcn/ui
- **バックエンド**: Laravel 12 (REST API) + PHP 8.5
- **DB**: PostgreSQL 18 (JSONB活用)
- **キャッシュ**: Redis 7
- **インフラ**: Docker Compose (ローカル) / AWS (本番)
- **AIチャット**: Gemini 3.1 Flash-Lite (システムプロンプト方式)

## 各ページ詳細

### 1. ログイン（管理画面）

- メールアドレス + パスワード
- セッション管理（JWT or Cookie）
- パスワードリセット

---

### 2. ダッシュボード

- 登録ユーザー数（総数・直近推移）
- 店舗数
- 口コミ件数
- AIチャット利用状況（日次・月次）
- 直近のアクティビティログ

---

### 3. ユーザー一覧

- テーブル表示（ページネーション・検索・フィルタ）
- 表示項目：
  - ユーザーID
  - LINE表示名 / LINEアイコン
  - Recta2プロフィール情報（入力済みなら）
  - 登録日時
  - 最終ログイン日時
  - 口コミ投稿数
  - ステータス（有効 / 停止）

---

### 4. ユーザー詳細

- プロフィール情報の閲覧（LINE情報 + Recta2入力情報）
- 口コミ投稿履歴
- チャット利用履歴（件数のみ or ログ）
- アカウント停止 / 有効化

---

### 5. 店舗一覧

- テーブル表示（ページネーション・検索・フィルタ）
- 表示項目：
  - 店舗ID / 店舗名
  - エリア / 最寄り駅
  - 業種（キャバクラ / ラウンジ / クラブ 等）
  - 公開ステータス（公開 / 非公開 / 下書き）
  - 口コミ件数 / 平均評価
  - 登録日 / 更新日

---

### 6. 店舗作成 / 7. 店舗編集

同一フォーム。入力フィールド：

#### 基本情報
- 店舗名
- エリア / 住所 / 最寄り駅
- 業種カテゴリ
- 営業時間
- 電話番号
- 公式サイトURL

#### 給与・待遇
- 時給（最低〜最高）
- 日給目安
- バック率（ドリンク / 指名 / 同伴 等）
- 保証給の有無・金額
- ノルマの有無・内容

#### 店舗の特徴・タグ
- 特徴タグ（例：「未経験歓迎」「終電上がりOK」「日払いあり」等）
- 自由テキストの紹介文
- お店の特徴テキスト

#### メディア
- 店舗画像（複数アップロード、並び替え可）
- 動画URL

#### 体入（体験入店）情報
- 平均時給
- 体入時給
- 面接可能時間
- 当日体入可否

#### 店舗分析
- 経験レベル（スライダー）
- 雰囲気（スライダー: 落ち着き↔賑やか）
- キャストスタイル（美人系/かわいい系/グラマー系/ナチュラル系の比率）
- 経験者:未経験者 比率
- 客層年齢分布
- 客層の飲み方

#### 面接・採用
- 面接時の服装アドバイス・Tips
- ドレスコード
- 採用基準
- 面接サポート対話（スタッフ/ユーザーの対話形式）

#### 必要書類
- 書類リスト（動的追加）
- 補足メモ

#### 勤務スケジュール
- 営業時間・定休日・シフト情報

#### 直近の採用実績
- 月ごとに追加（作成日ベース）: 月名・採用人数・採用例
- 直近合計テキスト

#### 人気の特徴
- 特徴リスト
- 他店ヒント

#### シャンパンメニュー
- **画像アップロードで管理**（テーブル入力ではない）

#### 送り・交通サポート
- **画像アップロードで管理**（送りエリアマップ等）

#### アフタースポット・同伴スポット
- スポット（動的追加: 店名+ジャンル+距離）

#### Q&A
- よくある質問と回答（動的追加）

#### スタッフコメント
- スタッフ名・役職・コメント・サポート内容

#### プレビュー
- エンドユーザーから見た表示をプレビューできるボタン

> **注**: データ構造は厳密でなくてよい。柔軟なJSON/JSONB型で保存し、フロントで表示を組み立てる方針でもOK。
> **削除した項目**: 駅からの道順（不要）、応募パラメーター（不要）

---

### 8. 口コミ一覧・モデレーション

- 全口コミの一覧表示
- フィルタ：店舗別・評価別・日付別・ステータス別
- 表示項目：
  - 投稿者（管理画面では内部ID/LINE名で特定可。公開は匿名）
  - 対象店舗
  - 評価（星）
  - 本文（一部プレビュー）
  - 投稿日時
  - ステータス（公開 / 非公開 / 削除済み）
- 操作：
  - 口コミ全文の閲覧
  - 公開 / 非公開の切り替え
  - 削除（論理削除）

---

### 9. AIチャット設定

#### ページ別プロンプト設定
AIチャットは3ページ（トップ・店舗一覧・店舗詳細）で使われており、ページごとに設定：
- AIチャットの有効 / 無効切り替え
- システムプロンプトの編集
- レスポンスのトーン設定

#### サジェストボタン設定
チャット入力欄上部の選択ボタンをページごとに管理：
- ボタンの追加・削除・並び替え
- 各ボタンの表示テキスト編集
- デフォルト:
  - トップ:「条件で探したい」「私の適正時給診断」「未経験で不安...」「今の流行りを知りたい」
  - 一覧:「ベスト3を厳選して！」「高時給なお店だけ」「ゆるく働けるお店」
  - 詳細:「私の査定額を聞く」「バックシステム詳細」「ノルマの有無を確認」

#### 利用制限（コスト対策）
- 1ユーザーあたりのチャット回数制限（日次 / 月次）
- 制限到達時のメッセージ設定
- IP単位の制限（未ログインユーザー対策）
- 全体の日次上限設定

#### 利用状況モニタリング
- チャット利用統計（日次 / 月次グラフ）
- ユーザー別利用回数
- APIコスト概算表示
- トークン使用量の推移

#### ナレッジ管理（将来検討）
- AIが参照する店舗情報・FAQの管理
- RAG用ドキュメントのアップロード

---

### 10. 管理ユーザー一覧

- 管理者アカウントの一覧
- 表示項目：名前・メールアドレス・ロール・最終ログイン・ステータス
- 操作：
  - 新規管理者の招待（メール送信）
  - ロール変更
  - アカウント無効化 / 削除
  - パスワードリセット

## LINE連携構成（確定方針）

### 構成図

```
┌─ ブラウザ ─────────────────────────────┐
│  Recta2サイト                           │
│  ・LINE Loginでログイン（OAuth 2.0）    │
│  ・口コミ閲覧・投稿（ログイン必須）    │
│  ・ユーザー情報入力・編集               │
│  ・店舗検索・AIチャット                 │
└─────────────────────────────────────────┘

┌─ 公式LINE ─────────────────────────────┐
│  ・友だち追加リンクでサイトから誘導     │
│  ・運営が1:1で手動チャット対応          │
│  ・LINE Official Account アプリで返信   │
└─────────────────────────────────────────┘
```

- **LIFF: 不要**（ブラウザ上で完結するため）
- 将来リッチメニューからサイトを開きたくなったら検討

### LINE Developersで必要なチャネル

| チャネル | 用途 |
|---------|------|
| LINE Loginチャネル | ブラウザからのOAuthログイン |
| Messaging APIチャネル | 公式LINEアカウント（1:1チャット + Webhook） |

※ 同一プロバイダー内で作成

### エンドユーザーのログインフロー

1. サイトで「LINEでログイン」タップ
2. LINE認証画面（OAuth 2.0）
3. 許可 → Recta2にリダイレクト
4. アクセストークンでユーザー情報取得（LINE ID・表示名・アイコン）
5. DB保存 → ログイン状態

### 口コミの閲覧・投稿制限

- 未ログイン → 口コミの一部のみ閲覧可、投稿不可
- ログイン済み → 全件閲覧可、投稿可
- 口コミは**匿名投稿**（LINE表示名は公開しない）

### ユーザー情報

- Recta2側でプロフィール入力フォームを用意
- LINE連携で取得した情報（LINE ID・表示名・アイコン）と紐づけて管理

## 必要なバックエンド基盤

現状フロントエンドのみのため、以下が必要：

### バックエンドAPI
- REST API or GraphQL
- 認証・認可ミドルウェア
- 画像アップロード（S3等のオブジェクトストレージ）

### データベース
- ユーザーテーブル（LINE連携情報含む）
- 店舗テーブル（柔軟なJSON構造でもOK）
- 口コミテーブル
- 管理ユーザーテーブル
- チャットログテーブル
- AIチャット設定テーブル

### 外部サービス連携
- **LINE Login API**: ユーザー認証
- **LINE Messaging API**: 公式LINEの1:1チャット + Webhook
- **Claude API / OpenAI API**: AIチャット
- **画像ストレージ**: S3 / Cloudflare R2 等
